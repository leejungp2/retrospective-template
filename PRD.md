# PRD.md — 일일/주간/연간 회고 웹앱 (MVP)

## 1. 제품 한 줄 정의
일일/주간/연간 회고를 **템플릿 글쓰기 / 대화형**으로 쉽게 작성하고, 회고에서 나온 **Action Item을 자동 추적**해 “기록 → 실행”으로 이어지게 하는 웹앱.

## 2. 핵심 가치(3)
1) **작성 부담 최소화**: 5분 회고, 위저드형 템플릿, 대화형 코치  
2) **실행으로 전환**: Try/Future action → To-do(액션) 자동화 + 다음 회고에서 자동 체크  
3) **쌓일수록 인사이트**: 태그/검색 + 반복 문제/잘되는 요인 + 주간/연간 요약 리포트

## 3. 타깃 사용자(3)
- **개인 사용자(기록/성장형)**: 매일/매주 회고로 자기관리(감정/습관/업무)  
- **스타트업/프로덕트팀 구성원**: 주간 회고로 실행 계획을 잡고 개선  
- **회고 버디(친구/동료)**: 서로 회고를 공유하고 액션을 지켜봐주는 파트너

---

## 4. 목표 / 성공지표(KPI)
### 목표
- 회고 작성 습관 형성(작성 유지율)
- 회고 → 액션 실행률 상승
- 회고 기록 탐색 및 요약 리포트로 재사용 가치 제공

### MVP 성공지표(예시)
- D7 리텐션(또는 W2 유지율)
- 주간 회고 작성률
- Action Item 생성 비율, 완료율(다음 회고에서 완료 체크 비율)
- 공유 기능 사용률(버디 연결/공유 링크 생성)

---

## 5. 범위 정의 (MVP)
### Must (MVP 필수)
- 로그인/회원가입
- 회고 주기: 일일 / 주간 / 연간
- 회고 방법론 4개: **KPT, 4F, 5가지 질문, 4L**
- 입력 방식 2개: **템플릿화된 글쓰기(위저드)** + **대화형(코치)**
- 공통 플로우: 배경 설정 → 팩트 나열 → 분석(방법론별) → Action Item 도출
- Action Item 관리: 생성/수정/완료 체크 + 다음 회고 시작 시 자동 리마인드
- 태그(기본): 회고에 태그 붙이기 + 검색/필터(최소 텍스트 검색)
- 친구 공유: 링크 공유(보기) + 섹션/요약/액션만 공유 옵션 (최소 1단계)
- 모바일 우선 UI + PWA(가능하면) 또는 최소 “홈 화면 추가” 가이드
- 데이터 내보내기(최소): Markdown export (PDF는 추후)

### Should (MVP 이후 빠른 확장)
- Fact Inbox(기간 중 팩트 메모 수집함) + 회고 작성 시 자동 불러오기
- 댓글/피드백(공유 권한이 댓글 가능일 때)
- 주간/연간 자동 리포트(반복 Problem, Keep TOP, Action 완료율)
- 리마인더(이메일/푸시 중 하나 선택)
- 오프라인 초안 저장(로컬) + 자동 복구

### Could (장기)
- 고급 템플릿(Gibbs/Kolb/5Whys 등), 질문 라이브러리(번아웃/관계/학습)
- 인사이트 대시보드(기분 추이, 주제별 히트맵)
- 팀 플랜(B2B): 워크스페이스, 익명 집계, 권한/보안 강화
- 암호/잠금, 민감 항목 숨김(검색 제외), PDF export

### Out of Scope(초기 제외)
- 캘린더/외부 앱(노션/슬랙/지라) 연동
- 실시간 협업 편집
- 복잡한 추천/AI 자동 작성(추후 옵션)

## 6. 회고 방법론(템플릿) 정의
### 공통 설계 원칙: “한 가지 공통 스키마”
- 모든 회고는 동일한 공통 필드를 갖고, 방법론은 **UI 섹션/질문 구성만 다름**
- 공통 필드(모든 회고 공통)
  - period_type: daily / weekly / yearly
  - date_range: 시작~끝
  - context: 회고 배경(왜/무엇을)
  - facts: 팩트 키워드/메모/첨부(링크/사진 메타)
  - sections: 방법론별 섹션(구조화 블록)
  - action_items: 다음 행동(추적 단위)

### 6.1 KPT (Keep / Problem / Try)
- Keep: 유지할 것
- Problem: 아쉬운/문제
- Try: 다음에 시도할 행동(액션으로 전환)
- 추천 주기: 주간, (간단히 일일도 가능)
- 섹션: keep[], problem[], try[] (각 항목은 block/card)

### 6.2 4F (Facts / Feelings / Findings / Future action)
- Facts: 객관 사실
- Feelings: 감정
- Findings: 교훈/통찰
- Future action: 행동 계획(액션 전환)
- 추천 주기: 일일/주간
- 섹션: facts[], feelings[], findings[], future_action[]

### 6.3 5가지 질문
- 잘한 것 / 잘못한 것 / 배운 것 / 남은 의문점 / 개선점
- 추천 주기: 주간, 연간(분기)에도 변형 가능
- 섹션: did_well[], did_bad[], learned[], questions_left[], improvements[]

### 6.4 4L (Liked / Learned / Lacked / Longed for)
- Liked: 좋았던 것
- Learned: 배운 것
- Lacked: 부족했던 것
- Longed for: 바라는 것
- 추천 주기: 주간
- 섹션: liked[], learned[], lacked[], longed_for[]

## 7. 입력 방식(2종) + 공통 플로우
### 7.1 템플릿화된 글쓰기(위저드)
- 단계 진행: (1) 배경/기간 (2) 팩트 (3) 섹션별 작성 (4) 액션 도출 (5) 요약
- 각 단계는 “최소 입력”을 강제하지 않되, 다음 단계로 넘어가기 쉬워야 함(작성 부담 최소)

### 7.2 대화형(코치 모드)
- 질문을 순차적으로 던져 답을 수집
- “짧게/깊게” 토글(질문 3~5개 vs 8~12개)
- **대화 결과는 반드시 구조화 블록으로 변환되어 저장** (검색/리포트/공유를 위함)
- 종료 시: 요약 + 섹션별 카드 자동 생성 → 사용자가 카드 편집

### 7.3 공통 플로우(모든 방식 동일)
1) 배경 설정: 회고 이유/기간/대상(프로젝트/역할/주제)  
2) 사실 기반 나열: 키워드 중심 팩트 + 첨부(링크/사진 메타)  
3) 내용 분석: 방법론 섹션에 카드(블록) 입력  
4) Action Item 도출: 행동 문장 템플릿 적용 + 다음 회고에서 자동 체크

---

## 8. Action Item 설계(실행률을 위한 최소 규칙)
- 회고에서 “Try/Future action/개선점”을 **Action Item으로 전환**
- Action Item 필드(최소)
  - title(행동 문장), due_date(optional), frequency(optional), status, success_criteria(optional)
- 다음 회고 시작 시 “지난 Action Item 3개” 자동 노출 + 체크 유도
- 완료 체크는 logs로 남김(action_item_logs)

---

## 9. 친구 공유 기능(민감성 고려)
### 공유 단위
- 전체 회고 / 특정 섹션만 / 요약만 / 액션만

### 권한(최소 1단계)
- 보기 전용(View)
- 댓글 가능(Comment) — Should로

### 공유 방식(MVP)
- 공유 링크 생성(토큰 기반) + 만료 기간 옵션(없음/7일/30일 중 하나)

---

## 10. IA(정보구조) & 주요 화면
### IA
- Home(Dashboard)
  - 오늘/이번 주 회고 CTA
  - 최근 회고 리스트
  - Action Item 상태 요약
- New Retrospective
  - (1) 기간/방법론 선택
  - (2) 입력 방식 선택: 위저드 / 대화형
- Retrospective Editor
  - 위저드 단계 UI 또는 대화형 UI
- Retrospective Detail(리뷰)
  - 섹션 카드, 요약, Action Item, 태그
- Action Items
  - 리스트/필터/상태 변경
- Share
  - 공유 범위 선택 + 링크 생성
- Settings
  - 계정/내보내기/리마인더(후순위)

---

## 11. BM(비즈니스 모델)
### Freemium(개인)
- Free
  - 기본 템플릿 4개, 저장 제한(예: 월 20개) 또는 리포트 제한
  - 기본 검색/태그
- Pro(월/연)
  - 고급 리포트(월/연 자동 요약), 고급 검색/필터
  - 내보내기(PDF/Markdown), 잠금/민감 항목 숨김
  - 고급 템플릿/질문 라이브러리, 멀티 디바이스 동기화 강화

### Team(B2B, 옵션)
- 워크스페이스, 권한, 익명 집계 리포트, 액션 오너 지정, 보안 강화
